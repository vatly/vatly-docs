# Subscription plans

Subscription plans are recurring products that enable you to create different subscription plans for your customers to choose from. On this page, we'll dive into the different subscription plan endpoints you can use to manage your subscription plans programmatically. We'll look at how to query, create, update, and delete subscription plans. {{ className: 'lead' }}

## The subscription plan model

The subscription plan model contains all the information about the subscription plans you create, including the name, description, and price.

### Properties

<Properties>
  <Property name="id" type="string">
    Unique identifier for the subscription plan.
  </Property>
  <Property name="name" type="string">
    The name of the subscription plan.
  </Property>
  <Property name="description" type="string">
    A detailed description of the subscription plan.
  </Property>
  <Property name="price" type="number">
    The price of the subscription plan.
  </Property>
  <Property name="interval" type="string">
    The billing interval (e.g., 'month', 'year').
  </Property>
  <Property name="status" type="string">
    The current status of the subscription plan (e.g., 'active', 'inactive').
  </Property>
</Properties>

## List all subscription plans {{ tag: 'GET', label: '/v1/subscription-plans' }}

<Row>
  <Col>

    This endpoint retrieves a list of all subscription plans.

    ### Optional query parameters

    <Properties>
      <Property name="limit" type="integer">
        The number of subscription plans to return (default: 10, max: 100).
      </Property>
      <Property name="startingAfter" type="string">
        A cursor for use in pagination. Returns results after this plan ID.
      </Property>
      <Property name="endingBefore" type="string">
        A cursor for use in pagination. Returns results before this plan ID.
      </Property>
    </Properties>

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="GET" label="/v1/subscription-plans">

    ```bash {{ title: 'cURL' }}
    curl -X GET https://api.vatly.com/v1/subscription-plans \
      -H "Authorization: Bearer live_your_api_key_here"
    ```

    ```php
    $vatly = new Vatly\Api\VatlyApiClient;
    $vatly->setApiKey($key);

    $subscriptionPlans = $vatly->subscriptionPlans->list();
    ```

    </CodeGroup>

    ```json {{ title: 'Response' }}
    {
      "data": [
        {
          "id": "plan_abc123",
          "name": "Basic Plan",
          "description": "Our starter plan with essential features",
          "price": 9.99,
          "interval": "month",
          "status": "active"
        },
        // More subscription plans...
      ],
      "has_more": false
    }
    ```

  </Col>
</Row>
